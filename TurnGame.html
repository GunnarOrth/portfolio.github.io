Turn Based Strategy Game In Java

The HotCiv game was a group project done over the course of a semester meant to teach us Java, as well as how to use certain tools in Java look factories and strategies. The basic principle is that each turn a player can just to move an attacker or build a city, and then the next turn the opposing player can do the same, the end goal being for one person to destroy the other player's cities. We created various strategies and factories however for various different ways for the game to function. We used interfaces for the strategies to more easily create implementations of said strategy. Each factory is made of strategies, each being a different version of the HotCiv game.

Overall, it was a very good experience to first learn Java and all the different things you can do with it. It was also good to see why using strategies and factories are necessary in larger programs. In the above video, my teammate Tej recorded a video showcasing the game.

What each strtegy does

  • World Layout: Determines what type of environment each tile is
  • World Aging: Determines the speed of world aging and start of aging
  • Winning: Determines what is necessary to win the game
  • Unit Production: Determines what item the unit will produce
  • Unit Action: Determines the actions a unit will perform
  • Attack: Determines if an attack is able to be carried out depending on attributes in the implementation
