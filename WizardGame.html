Wizard Game
Unreal Engine/C#

In this page I will document my progress on a game made in unreal engine. My idea for the game is for it to be an open world action adventure game similar to the legend of zelda breath of the wild. However, I want to put a heavier focus on spells and potions the player can learn throughout their adventure to aid them throughout many different environments, puzzles, quests and NPC's they meet. Each spell is unique and can be used to solve puzzles and fight unique enemies the player will face. I would also like to implement some sort of multiplayer later on. This will be my most ambitious project yet so updates may take time.

As of now, I have created some basic prototypes of the most important parts to an action adventure game including, spells, inventory, AI, and interaction. All of the assets like animations, images, and models are free assets and are only placeholders to show how things work in the game, and will be changed later on. There is much more work that needs to be done but this progress gives me a good stepping stone to branch out into any area I will work on in the future.

Spells

Spells are cast using Unreal Engines advanced input system. First, the spells are added to an array of spells in the third person characters construct event. Then, holding down right click, a user interface will show up that will highlight what section of the weapon wheel you are in. Once you are in the selected section of the wheel, moving outside of the wheel will allow you to access specific spells of that category of spells. Currently there are 8 sections and 8 spells within each section. One left click is pressed, it will spawn that spell actor from the array and call the spells event using a spell blueprint interface. Currently there are only two spells. One projectile that fires from the player and one line trace that draws a line to the center of the screen. These two spells will be the prototype for the majority of the other spells

Tactical Abilities

Tactical Equipment is somewhat similar to spells. They are also held in an array that will create the desired equipment actor based on what index is selected using the scroll wheel. However, tactical equipment is picked up and shown in the users inventory, or crafted later on. The tacticals also use a blueprint interface to execute their behavior. Using the scroll wheel or using a tactical will change the UI. Tacticals will be implemented later on to be things like bombs and health potions.

Third Person Character's Q event that will create the actor at the tactical socket in the character's mesh if there are any in the user's inventory. It will then call said actor's interface.

EquipScroll will change the tactical selected and change the UI to show the amount of the tactical and which one is chosen.

Inventory & UI

The inventory can be opened using tab or the 'T' key. In the inventory UI there is a seperate widget for resources, which are items you can gather throughout the map but are only used to craft tactical equipment. The items widget will show you what tacticals you have and how many of them you have. You may also craft more of an item by clicking its button and it will remove those resources and add an item. These are the items you can access as tacticals. The armor widget displays what armor you have and what body part it is used for. By clicking on an armor pieces button you can equip it. The quests widget currently contains nothing but will show progress on quests. The spells widget also is not implemented but will be where you can level up spells so that you may cast them. When the inventory UI is constructed it will go through the inventory arrays and create child widgets and add them to there respective parents within the inventory UI.

Interaction

Interaction is done by using unreal engines advanced input action to press 'E'. When E is pressed the engine will loop over overlapping actors and check if they implement the interactable interface. If they do they will break the loop and call the interact function. The interact interface will be used on many more things like doors, lights, etc...

Enemy AI

The basic enemy AI is created by creating a navmesh in the environment and then creating a navmeshagent to traverse the terrain. I created a basic navmeshagent and an AI controller to control its movement with a behavior tree. The behavior tree will determine its movements based on selectors that will decide which sequence to execute, and sequences that will run through different tasks for the AI to execute. In the basic enemy I created, the behavior tree will first check if the Ai is meant to be attacking or if it's on patrol. If it's attacking, it will then check if it is wielding a weapon, if not it will wield a weapon. If it is will run the chase player sequence which includes chasing the player, stopping to attack, and clearing and adding focus to the player to make the movement more realistic. If it's on patrol it will follow a spline created in the navmesh.

Menus

Currently there is a main menu, a pause menu, and an options menu. The main menu has a single player button that will open an instance of the third person character level, a multiplayer button, an options button that will take you to the options menu, and a quit button that will close the game. The pause menu has a return to menu button, resume game, and options menu button. The options menu gives you access to change the window mode, graphics, resolution and Vsync, as well as an apply settings and return to menu button.
